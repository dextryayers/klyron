# v0.1.0 -- Initial Release

**Release Date**: 2026-07-16

Welcome to the initial public release of Klyron! This release marks the first stable milestone of the Universal Polyglot Runtime & Developer Platform.

## Features

### Runtime

- Multi-engine support: V8 (default), Boa, QuickJS, JavaScriptCore
- Permissions-based sandbox with granular `--allow-*` flags
- Cross-platform support: Linux (x86_64, ARM64), macOS (x86_64, ARM64), Windows (x86_64)
- ES modules (ESM) and CommonJS compatibility
- TypeScript compilation (no separate build step)
- Node.js compatibility layer for built-in modules (fs, path, http, etc.)

### Package Manager

- Fast content-addressable package store
- pnpm-compatible symlinked node_modules
- Lockfile generation (`klyron.lock`) with SRI integrity hashes
- Workspace support for monorepos
- Multi-registry support (Klyron registry + npm registry)
- Package signing with GPG

### Dev Server

- Hot Module Replacement (HMR) with WebSocket communication
- Auto-detection of React, Vue, Svelte, and Laravel projects
- Built-in proxy configuration for API backends
- HTTPS with auto-generated certificates
- CSS processing with PostCSS and CSS modules

### Build System

- Production bundler with tree-shaking
- Multi-format output: ESM, CJS, IIFE, WASM
- Minification and sourcemaps
- Code splitting and dynamic imports
- Asset hashing and pipeline (images, fonts, CSS)

### Test Runner

- Built-in assertion library with common matchers
- Coverage reporting (lcov, HTML, text, clover)
- Watch mode with file change detection
- Adapter for Vitest, Jest, PHPUnit, and pytest
- Snapshot testing
- Parallel test execution

### Framework Adapters

- **Laravel**: Vite integration, Artisan proxy, Blade compilation, Inertia SSR
- **Rails**: Propshaft, importmaps, Turbo/Hotwire
- **Django**: Whitenoise, Daphne ASGI, template compilation
- **Next.js**: React Server Components, App Router

### Plugin System

- WASM-based plugin architecture
- Lifecycle hooks: on_load, on_unload, on_transform, on_request, and more
- Plugin permission model with granular controls
- Host API for logging, config, file access, and HTTP

### CLI

- `klyron run` -- Execute files with sandbox and engine selection
- `klyron dev` -- Start dev server with HMR
- `klyron build` -- Bundle for production
- `klyron install` -- Install packages
- `klyron test` -- Run tests with coverage
- `klyron create` -- Scaffold projects from templates
- `klyron workspace` -- Manage monorepo workspaces
- `klyron publish` -- Publish packages to registry
- `klyron add` -- Install adapters

## Known Issues

### Runtime

- `--allow-ffi` flag is not yet implemented on Windows
- V8 engine requires approximately 200MB of disk space for compiled cache
- QuickJS engine does not support `SharedArrayBuffer`
- JavaScriptCore engine does not support `fetch()` API on Linux

### Package Manager

- Install performance on large monorepos (>100 packages) can be slow; optimization in progress
- Git dependencies (`git+https://`) are read-only; private git repos require SSH agent setup
- Registry search (`klyron search`) is not yet implemented

### Dev Server

- Laravel SSR with Inertia may have hydration warnings in development mode
- WebSocket reconnection on network interruption is not yet handled gracefully
- Custom HMR handlers for Lit and Solid use generic fallback (full reload)

### Build System

- WASM output target is experimental; only basic use cases are supported
- Sourcemaps for CSS are not yet generated
- Bundle analysis (`--analyze`) is terminal-only; no interactive viewer

### Test Runner

- PHPUnit adapter requires PHP 8.1+ and `phpunit/phpunit` installed via Composer
- pytest adapter does not support `pytest-xdist` for distributed execution
- Snapshot updates (`--update-snapshots`) overwrite files without confirmation

### Plugin System

- Plugin hot-reloading requires a full Klyron restart
- No plugin debugger or performance profiler
- Plugin ABI version 0.1.0 is not guaranteed to be backward compatible

## Installation

```bash
cargo install klyron --version 0.1.0
```
