# `klyron test`

Run tests with a built-in assertion library, coverage reporting, and multi-framework support.

## Usage

```bash
klyron test [options] [filter]
```

## Description

The `test` command discovers and runs test files in your project. It includes a built-in assertion library, code coverage via source instrumentation, and adapters for popular test frameworks. Test files are identified by naming convention: `*.test.{js,ts}`, `*.spec.{js,ts}`, `*.test.php`, `*_test.py`, or files in a `__tests__/` directory.

## Options

| Option | Description |
|---|---|
| `--watch` | Watch mode: re-run tests on file changes |
| `--coverage` | Generate code coverage report |
| `--coverage-threshold <pct>` | Fail if coverage is below threshold |
| `--coverage-report <format>` | Output format: lcov, html, text, clover |
| `--reporter <reporter>` | Test reporter: dot, spec, tap, junit, json |
| `--timeout <ms>` | Per-test timeout (default: 5000) |
| `--parallel` | Run tests in parallel |
| `--shard <n/m>` | Run only a shard of the test suite |
| `--bail` | Stop on first failure |
| `--update-snapshots` | Update snapshot files |
| `--filter <pattern>` | Run tests matching a pattern |

## Built-in Assertion Library

Klyron includes a built-in assertion library that works without any imports:

```typescript
import { test, expect, describe } from "klyron:test";

describe("Math", () => {
  test("addition", () => {
    expect(1 + 1).toBe(2);
    expect(1 + 1).not.toBe(3);
  });

  test("object equality", () => {
    expect({ a: 1 }).toEqual({ a: 1 });
    expect([1, 2, 3]).toContain(2);
  });

  test("async", async () => {
    const result = await fetchData();
    expect(result).toBeDefined();
    expect(result.length).toBeGreaterThan(0);
  });
});
```

### Available Matchers

| Matcher | Description |
|---|---|
| `toBe(value)` | Strict equality (`===`) |
| `toEqual(value)` | Deep equality |
| `toBeNull()` | `=== null` |
| `toBeDefined()` | `!== undefined` |
| `toBeTruthy()` | Truthy check |
| `toBeFalsy()` | Falsy check |
| `toBeGreaterThan(n)` | `> n` |
| `toBeLessThan(n)` | `< n` |
| `toContain(item)` | Array/string inclusion |
| `toMatch(regexp)` | Regex match |
| `toThrow()` | Expects function to throw |

## Multi-framework Support

Klyron detects and delegates to installed test frameworks:

| Framework | Detection | Notes |
|---|---|---|
| Vitest | `vitest` in dependencies | Full compatibility |
| Jest | `jest` in dependencies | Via adapter |
| PHPUnit | `phpunit/phpunit` in composer.json | Native PHP testing |
| pytest | `pytest` in dev dependencies | Python test support |

Framework configuration files (e.g., `vitest.config.ts`, `jest.config.js`) are respected automatically.

## Examples

### Run all tests

```bash
klyron test
```

### Watch mode

```bash
klyron test --watch
```

### With coverage

```bash
klyron test --coverage --coverage-threshold 80
```

### Run a specific test file

```bash
klyron test src/utils.test.ts
```

### Run tests matching a pattern

```bash
klyron test --filter "math"
```

### Parallel execution

```bash
klyron test --parallel
```

### PHPUnit integration

```bash
klyron test tests/Feature/
```

## Coverage

Coverage is generated by instrumenting source files during test execution. Reports can be output in multiple formats:

```bash
klyron test --coverage --coverage-report lcov --coverage-report html
```

The HTML report is written to `coverage/` and provides an interactive viewer.

## Configuration

Test configuration can be set in `klyron.json`:

```jsonc
{
  "test": {
    "include": ["src/**/*.test.{ts,js}"],
    "exclude": ["node_modules", "dist"],
    "setupFiles": ["src/test-setup.ts"],
    "environment": "node",
    "globals": true,
    "coverage": {
      "enabled": true,
      "threshold": 80,
      "include": ["src/**"],
      "exclude": ["src/**/*.test.*"]
    }
  }
}
```
